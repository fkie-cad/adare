Create an Environment

An environment defines the vagrant box to be used combined with metadata about the guest OS. Additionally it can contain a list of installations to run before the experiment. This data is specified in a YAML file we call the environment configuration file.

To load an environment into an project, we can run the adare environment load <environment_name> command. Therefore the environment configuration file must be located in the environments directory within the project directory. Notably

All options allowed in the environment configuration file are listed in the table below:

options of the environment configuration file

Field

Type

Description

name

str

The name of the environment

vagrantbox

str

Either the name of a local vagrant box OR the name of an vagrant box from vagrant cloud (format: user/boxname)

os

dict

information about the guest OS

os.os

str

The OS to use (e.g. windows 10, ubuntu 20.04)

os.platform

str

the guest os platform (e.g. windows, linux)

os.distribution

str

The OS distribution to use (e.g. Home, Pro, Enterprise)

os.version*

str

The OS version to use (e.g. 10, 16.04)

os.architecture*

str

The OS architecture to use (e.g. x86, x64)

os.language*

str

The OS language to use (e.g. en-US, en-GB)

os.details*

str

further details about the OS

postsetupinstallations*

dict

A list of installations to run before the experiment

postsetupinstallations.name

str

The name of the installation (e.g. install python3)

postsetupinstallations.command

str

The command to run (e.g. apt-get install -y python3)

postsetupinstallations.description*

str

A description of the installation

postsetupinstallations.cwd*

str

The directory to run the command in

postsetupinstallations.shell*

bool

value to indicate if the command should be run in a dedicated shell (default: false)

Note

Fields marked with an asterisk (*) are optional.
