Action File
************

The action file is a python file containing all actions that mimic user behaviour.
To create a skeleton for a new action file, as shown below, run the following command::

    adare exp create <experiment_name> --env <environment_name>

.. code-block:: python

    from pathlib import Path
    import sys

    from guiautomation.Experiment.Experiment import Experiment
    from guiautomation.run import run

    import logging
    log = logging.getLogger(__name__)


    class TestExperiment(Experiment):
        description = 'place the description of your experiment here'

        def __init__(self):
            super().__init__()

        def prepare(self):
            pass

        def run(self):
            log.info(f'experiment {type(self).__name__} started')

            # place the code to execute some stuff here

            log.info(f'experiment {type(self).__name__} done')

    if __name__ == '__main__':
        if len(sys.argv) < 2:
            print('missing file for config path')
            exit(-1)
        run(TestExperiment, config_file=Path(sys.argv[1]))

The created skeleton is a python file containing a class named by the experiment name, which inherits from the class ``experiment``.
It contains multiple three parts, which should be adjusted to the needs of the experiment.
The first one is the description attribute of the class, which should contain a telling description for the experiment.
The second one is the ``prepare`` method, which is called before the experiment is executed and can be used to prepare the experiment (e.g. create necessary files).
The third one is the ``run`` method, which contains the action done in the experiment.
This can include executing commands in the shell, clicking on specific text or icons.
The gui actions are performed using GuiBot, which provide methods to find matches of images or text on the screen.
These have the type ``guibot.match.Match`` and contain the position of the match and the image or text that was matched.
Additionally those are used within GuiBot methods, which perform actions, such as clicking on match.
Here is a list of all available methods as well as the corresponding description taken from the `GuiBot source code <https://github.com/intra2net/guibot/blob/fb3c26ce5b678aee76b1d93c7d76017122f04a64/guibot/region.py>`_:
For further details on the methods as well as their parameters, please refer to the source code.

To find matches of images or text on the screen, one can use the methods provided by GuiBot.
However to simplify usage, the ``Experiment`` class provides some methods that wrap GuiBot functions for more robust text and image finding.
For the find methods it is needed that the file is placed within the ``img`` directory within the experiment directory, which is also created when creating the skeleton.

.. py:function:: find(image_name: str, minimal_similarity: float = 0.6, similarity_steps. float = 0.1) -> list[guibot.match.Match] or None

        Finds the image in the current screen and returns a list of matches or None if no match was found.
        Therefore, creates a chain of searches starting with a 90% similarity and decreasing the similarity by 10% until the image is found.
        If the image is not found with a similarity of the minimal similarity parameter the search is aborted and None is returned.

        :param image_name: the name of the image to find
        :type image_name: str
        :param minimal_similarity: the minimal similarity to find the image with
        :type minimal_similarity: float
        :param similarity_steps: the steps to decrease the similarity by
        :type similarity_steps: float

        :return: a list of matches or None if no match was found


.. py:function:: find_text(text: str) -> list[guibot.match.Match] or None

        Finds the text in the current screen and returns a list of matches or None if no match was found.

        :param text: the text to find
        :type text: str
        :return: a list of matches or None if no match was found



Additionally to that the ``Experiment`` class provides a few methods to perform other actions.
A list of those methods can be found below.

.. py:function:: exec_shellcommand(command: list, cwd=None):

        Executes the given command in the shell.

        :param command: the command to execute
        :type command: list
        :param cwd: the working directory to execute the command in
        :type cwd: str

In some cases it might be necessary to save certain data during the action phase that may be needed later on in the preformed tests (such as timestamps).
Therefore the ``Experiment`` class offers the two methods described below.

.. py:function:: save_variable(name: str, value: str)

        Saves the given value under the given name in the variables file.

        :param name: the name to save the value under
        :type name: str
        :param value: the value to save
        :type value: str

.. py:function:: save_time(timestamp_var_name: str)

        Saves the current timestamp under the given name in the variables file.

        :param timestamp_var_name: the name to save the timestamp under
        :type timestamp_var_name: str


Action File Example
###################

One example for an action file, which deletes a file is shown below.

.. code-block:: python

    from pathlib import Path
    import sys

    from guiautomation.Experiment.Experiment import Experiment
    from guiautomation.run import run
    from guibot.guibot import GuiBot

    import logging
    log = logging.getLogger(__name__)


    class deletefile(Experiment):
        description = "Delete file from home folder to Trash Bin (Ubuntu)"
        guibot: GuiBot = None

        def __init__(self, *args, **kwargs):
            super().__init__(*args, **kwargs)

        def prepare(self):
            file = "C:/Users/vagrant/Documents/testfile"
            log.info(f'create the file {file}.')
            with open(file, mode="w") as f:
                f.write("GEHEIM!")
            log.info(f'file {file} created.')

        def run(self):
            guibot = self.guibot

            log.info(f'experiment {type(self).__name__} started')

            match = self.find('files.png')
            if match:
                guibot.click(match[0])
                guibot.idle(5)
            else:
                log.error('files.png button does not exist')
                return 'failed'

            match = self.find_text('Documents')
            if match:
                guibot.click(match[0])
                guibot.idle(5)
            else:
                log.error('documents text does not exist')
                return 'failed'

            match = self.find_text('testfile')
            if match:
                guibot.click(match[0])
                guibot.idle(5)
                guibot.press_keys(guibot.dc_backend.keymap.DELETE)
                self.save_time("DELETIONDATE")
            else:
                log.error('testfile text cant be found')
                return 'failed'

            log.info(f'experiment {type(self).__name__} done')
            return self.status


    if __name__ == '__main__':
        if len(sys.argv) < 2:
            print('missing file for config path')
            exit(-1)
        run(deletefile, config_file=Path(sys.argv[1]))
